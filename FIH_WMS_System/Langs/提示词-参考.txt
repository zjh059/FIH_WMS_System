角色与背景
你是我的编码助手。我们正在开发一个名为 FIH_WMS_System 的 C# WinForms 智能仓储管理系统。目前我们正在处理**多语言切换（简中 zh-CN、繁中 zh-TW、英文 en-US）**功能。

当前采用的技术方案与逻辑

核心机制：放弃原生 .resx，采用全局 JSON 字典 + 动态遍历翻译方案。通过 LanguageHelper.cs 工具类读取对应语言的 JSON 包，并通过 TranslateForm() 递归刷新窗体控件文本。

热切换：主界面的 btnSetupLangs 按钮可一键循环切换（简->繁->英）。切换时调用 Application.OpenForms 遍历并动态翻译已打开的窗口，无需重启软件。子窗口需在自身的 _Load 事件末尾主动调用 TranslateForm(this)。

防丢失自动抓取：当开启 EnableAutoRecord = true 时，如果在非简中模式下遇到字典中缺失的词条，系统会自动将其捕获并合并写入 [语言]_待翻译.json 文件中，防止漏翻和数据覆盖。

双键合一结构（终极方案）：为了解决“英文切繁体时系统因找不到原版简体字而报错漏抓”的痛点，我们在非简中的 JSON 字典（如 en-US.json 和 zh-TW.json）中采用了双键映射机制。即将“简体中文”和“已翻译的语言”同时作为 Key，指向同一个最终翻译结果。

已解决的痛点与开发注意事项

动态变量拼接坑：绝对不能将带有动态变量（如人名、角色名）的整句长文本作为翻译 Key（例如 "当前登录：【t】 (管理员)"）。必须在代码中将静态短语（当前登录：）和动态变量拆开，分别翻译后再拼接（如在 MainForm_Load 中处理）。扫描翻译前需将 this.Text 还原为纯静态基础词，翻译后再拼接还原，以防长句被错误抓取。

JSON 语法严格：遇到字典抓取失效时，90% 概率是因为手动编辑 JSON 时漏了逗号或双引号。LanguageHelper.LoadLanguage 中已加入 try-catch 进行格式报错弹窗拦截。
